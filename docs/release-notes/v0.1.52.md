# v0.1.52 - Learning Systems Completion

v0.1.52 closes three learning-system gaps after the v0.1.51 corrections loop.

The GNN auto-trainer is now enabled by default with bounded cost rails still in
place: 1h minimum throttle and 5min runtime cap. Trigger thresholds now fit the
first few normal sessions: first run 100 -> 30 memories, new-memory trigger
50 -> 20, corrections 5 -> 3. New `archon learning gnn status` is available in
CLI and slash form.

Constellation drift detection can now bootstrap cold workspaces. Use
`archon constellation bootstrap --target memory|docs|session`, or let
`detect_drift` auto-bootstrap memory/docs targets on first use. Too-thin
sources return `DriftStatus::BootstrapPending` instead of failing as a missing
centroid.

`archon meaning` triplets now feed the GNN trainer. Triplets hydrate to
anchor/positive/negative embeddings, contribute a conservative auxiliary margin
loss, and show up in per-epoch loss breakdowns alongside trajectory quality and
EWC. Empty or unresolvable triplet paths no-op cleanly.
