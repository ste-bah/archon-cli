# v1.1.0-beta.1 - Provider Runtime Governance

v1.1.0-beta.1 is a beta release for the provider runtime and governed agent
evolution work. It moves provider behavior, permissions, sandbox routing, and
agent profile changes into durable Cozo-backed governance paths.

Provider execution now emits durable runtime evidence. Archon records provider
requests, successes, failures, fallback decisions, auth-profile health, status
snapshots, and rate-limit windows in CozoDB. The provider status surfaces can
show live checks, recent limits, fallback health, and JSON snapshots for
diagnostics.

Codex support now includes the app-server runtime path. Archon can use direct
Codex routing or a JSON-RPC app-server transport over WebSocket or stdio.
Configured WebSocket endpoints accept `ws`, `wss`, `http`, and `https` schemes
for compatibility. Codex app-server model discovery and rate-limit updates are
recorded instead of being hardcoded or process-local only.

Anthropic Claude Code compatibility remains protected. The Claude OAuth/API-key
path keeps the existing spoofing behavior, and contract tests cover the identity
headers, metadata, system blocks, and provider-injection boundaries that agents
depend on.

Agent evolution is now governed by runtime evidence. Archon records session
turns and incidents to a Cozo-backed performance ledger, generates proposals
from observed patterns, supports shadow evaluation before activation, applies
approved profile overlays, records profile versions, and supports rollback,
history, status, digest, and report commands.

Memory promotion now goes through Archon's memory system. Candidate memories are
deduplicated, reviewed, and promoted into the graph-backed memory store rather
than written as markdown memory files.

Permission and sandbox governance now run before tool execution. The real agent
preflight path calls the permission checker before tool lookup and routes Bash
execution through audited sandbox backends. Docker, SSH, OpenShell, logical,
and disabled backends are documented with status, doctor, explain, and test
surfaces. The default `sandbox.mode = "risky"` keeps normal host-side coding
tools under permission preflight while routing Bash/Shell through the selected
real backend; `sandbox.mode = "all"` is the strict compatibility mode.

Runtime hooks now cover the provider and agent lifecycle. Hook events exist for
provider resolution, prompt build, agent run, tool calls, learning events, and
agent profile application. Behavior-changing hook output is ignored on the new
runtime lifecycle path so hooks cannot bypass spoofing, permission gates, or
sandbox controls.

Setup documentation and dependency tooling now cover the new sandbox backends.
`scripts/install-system-deps.sh` keeps the default required build/PDF/OCR path
small, and adds explicit `--with-docker`, `--with-openshell`, and
`--with-sandbox` modes for hosts that will use Docker or OpenShell isolation.
OpenShell users can also opt into per-user local gateway setup with
`--setup-openshell-gateway`; the installer refreshes OpenShell through
NVIDIA's official installer, starts/registers the local gateway service on
macOS/Linux, and keeps legacy `openshell gateway start` support as a fallback.
