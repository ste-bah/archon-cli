# v1.2.7

v1.2.7 fixes a resume-only Anthropic shape failure from sessions written by
older binaries and removes the last fake `200k` context-window assumptions from
provider metadata.

## Resume Repair

- Adds a sanitizer pass that merges consecutive user messages whose content is
  exclusively `tool_result` blocks. Anthropic requires all results for one
  assistant tool batch to live in the single immediately-following user
  message; older persisted sessions could split them across adjacent messages.
- Keeps mixed user content untouched. A user message containing text plus a
  tool result is deliberately not merged because the text may be semantically
  adjacent to that turn.
- Preserves the v1.2.6 write-time safety net:
  `ConversationState::add_tool_result` already batches adjacent tool results for
  newly-written state.
- Anthropic, Bedrock, and Vertex benefit through the existing shared
  Anthropic-shape sanitizer.

## Context Windows

- Adds a bundled `context.toml` catalog for model context-window limits.
- Loads optional user/project overrides from `~/.config/archon/context.toml`,
  `~/.archon/context.toml`, `<workspace>/.archon/context.toml`, and
  `<workspace>/.archon/context.local.toml`.
- Preconfigures Codex-auth GPT models with current OpenAI documented limits:
  GPT-5.5 and GPT-5.4 at 1,050,000 tokens; GPT-5.4 Mini and GPT-5.3 Codex at
  400,000 tokens.
- Preconfigures Claude paid-plan context from Anthropic support guidance:
  Opus 4.7 and Sonnet 4.6 use 500,000 tokens by default and 1,000,000 tokens
  when running through the Claude Code spoof identity path.
- Unknown third-party/local models now resolve to unknown instead of an invented
  200,000-token cap. Proactive compaction is skipped when the limit is unknown;
  reactive compaction on provider context-window errors still works.

No public API shape changes.
