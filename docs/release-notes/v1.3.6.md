# Archon v1.3.6 — Video Evidence Capture Fallbacks

## Video Evidence

- YouTube video+frame ingest now requests a frame-friendly MP4-oriented format
  by default, with `ARCHON_YTDLP_VIDEO_FORMAT` as the override.
- Caption capture can now run before local ASR when
  `[policy.video].allow_caption_capture = true`; failed or missing captions
  fall back to the configured ASR provider.
- Frame extraction can optionally fall back to Python/OpenCV when `ffmpeg`
  cannot produce frames from a local video.
- Local image/frame OCR can use RapidOCR first or as a fallback to Tesseract.

## Documentation

- Updated the video guide, policy reference, env-var reference, installation
  guide, and YouTube cookbook with the caption, RapidOCR, OpenCV, and
  frame-friendly `yt-dlp` paths.
