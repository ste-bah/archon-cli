# Archon v1.3.9 — Document Vector Store Acceleration

v1.3.9 moves document embeddings out of the hot Cozo write path while keeping
Cozo as the inspectable source of truth for documents, chunks, provenance,
index jobs, and queue state.

## Added

- RocksDB raw-vector store at `.archon/doc-vector-store`.
- Rust-HNSW snapshot compaction from persisted raw vectors.
- `archon docs vector-status` and `/docs vector-status`.
- `archon docs vector-migrate` and `/docs vector-migrate` for resumable
  legacy Cozo vector migration.
- `archon docs vector-compact` and `/docs vector-compact` for rebuilding
  semantic search snapshots without re-embedding.
- `ARCHON_DOC_VECTOR_STORE_DIR` override for the raw-vector store location.
- `ARCHON_DOCS_LEGACY_COZO_VECTOR_WRITE` compatibility flag for operators who
  need dual writes during a transition.
- Provider-aware indexing workers with optional multi-instance local fastembed
  via `ARCHON_DOCS_FASTEMBED_INSTANCES`.

## Changed

- New document indexing writes embeddings to RocksDB by default and updates
  Cozo chunk/queue status separately.
- Embedding cache reuse now comes from the RocksDB content-hash cache.
- Model-status output reports legacy Cozo rows, RocksDB raw-vector/cache rows,
  and Rust-HNSW snapshot state.
- Evidence Engine docs and diagrams now show the split persistence model.
- Local fastembed workers are capped to actual model-instance parallelism
  instead of sharing one mutexed model across several worker threads.
- Document indexing now uses longer bounded Cozo lock retries and avoids
  rewriting vectors when only the post-write status update needs retrying.
- Cached embedding reuse now bulk-writes cached vectors and bulk-updates
  chunk/queue status instead of marking each cached chunk through Cozo one row
  at a time.
- Cozo writes now share a guarded retry and cross-process sidecar lock helper,
  so document indexing, learning store opens, and world-model metadata writes
  coordinate instead of racing the SQLite-backed Cozo writer.
- TUI splash snapshots updated for v1.3.9.

## Migration

Existing Cozo vectors do not need to be discarded or re-embedded:

```bash
archon docs vector-status
archon docs vector-migrate --limit 1000 --batch-size 250
archon docs vector-migrate --after <last-chunk-id> --limit 1000
archon docs vector-compact
```

`vector-migrate` skips rows already present in RocksDB and prints the last
scanned chunk id for safe resume.
