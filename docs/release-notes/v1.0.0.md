# v1.0.0 - Audited Pipeline Runtime And Self-Calibration

v1.0.0 is the final completion release for the current Archon feature set.
Future feature changes are expected to move through major-version updates.

Built-in coding and research pipelines now write audited run bundles under
`<workdir>/.archon/pipelines/<session-id>/`. Each bundle contains a manifest,
checksum-protected state, append-only audit log, prompt records, per-agent
records, accepted outputs, retry attempt outputs, verification reports, and
trace exports. Resume now verifies the bundle before continuing and hydrates
completed agents from verified records.

New inspection surfaces:

- `archon pipeline verify <session-id> [--write-report]`
- `archon pipeline inspect <session-id>`
- `archon pipeline export-traces <session-id> [--out traces.jsonl]`

Pipeline completion now runs completion-integrity checking for the final
answer and records the summary/report id back into the audited bundle. LLM
stream errors now fail closed with a partial-output hash instead of silently
accepting truncated output.

Self-calibration now has a first shell surface. `archon self retrospective
<session-id>` reads persisted session activity JSONL, extracts a small set of
evidence-backed lessons, writes retrospective artifacts, and attempts to store
matching memories and LearningEvents. `archon self trust status` reports
domain-scoped self-trust, and `archon self plans inspect <session-id>` compares
stored plans with recorded outcomes.

Existing memory surfacing now records telemetry when relevant memories or
first-turn briefings are injected. v1.0.0 does not replace the memory briefing
path; it makes that path visible to later retrospective analysis.
