> **Note on versioning.** v0.1.37 shipped from origin/main as a Tokio fix
> while this Evidence Engine arc was in flight. v0.1.38 was tagged at the
> tip of the arc but the tag landed on a commit before CI was fully green
> on macOS; rather than force-move a published tag we bumped to v0.1.39
> on top of the CI fixes (registry counts, file-size whitelist, macOS
> `/var` Ōåö `/private/var` canonicalization). v0.1.39 is the first
> tagged build of the Evidence Engine arc with a green CI matrix.

The Evidence Engine work that ran across Phases 1-6 (document intelligence,
retrieval, game-theory, completion integrity, governed learning) is now
PRD-compliant at the code level per `docs/evidence-engine-compliance-matrix.md`
(54 DONE / 18 PARTIAL / 0 GAP), and the OAuth-authenticated full pipeline now
runs real LLM Tier 1 + specialist execution end-to-end.

## What's new since v0.1.36

### Five new crates (all wired through the CLI)

- `archon-knowledge` ŌĆö claim extraction, entity recognition, relations,
  source quality, contradiction scanning, hybrid retrieval. CLI: `archon kb`.
- `archon-provenance` ŌĆö W3C PROV chain hashing, JSON-LD export, traversal
  API. CLI: `archon prov {trace, export, verify}`.
- `archon-meaning` ŌĆö derived sample/contrastive-pair/triplet construction
  for downstream learning. CLI: `archon meaning {build, samples,
  contrastive, triplets, export}`.
- `archon-constellation` ŌĆö workspace centroid construction, scoring, drift
  detection. CLI: `archon constellation {build, score, drift, list}`.
- `archon-policy` ŌĆö TOML policy layer with `[policy.gametheory]`,
  `[policy.learning]`, `[policy.docs]` sections; default-deny on risky
  surfaces.

### Game-theory pipeline

- Full 12-tier routing spec at `.archon/specs/gametheory.yaml` (463 lines)
  covering all 84 game-theory agents.
- Tier 1 mandatory agents corrected to the canonical PRD set:
  `game-classifier`, `payoff-elicitor`, `strategy-space-enumerator`,
  `information-structure-mapper`.
- `archon gametheory` now exposes 11 subcommands: `run`, `list-runs`, `show`,
  `status`, `inspect`, `inspect-fingerprint`, `inspect-routing`, `replay`
  (with `--reclassify` and `--rerun-specialist`), `resume`, `list-agents`,
  `specimens`. Plus flags `--budget`, `--max-concurrent`, `--style`,
  `--classify-only`, `--kb`, `--spec-path`.
- 8 game-theory tools registered in `archon-tools`: `GameTheoryRun`,
  `GameTheoryStatus`, `GameTheoryListAgents`, `GameTheorySpecimens`,
  `GameTheoryInspect`, `GameTheoryReplay`, `GameTheoryClassify`,
  `GameTheoryCallSpecialist`.
- TUI gametheory tab + evidence browser + learning tab wired (load rows
  from persisted Cozo state).

### OAuth + identity-spoof execution

- `ANTHROPIC_API_KEY` values prefixed `sk-ant-oat-` (OAuth tokens issued by
  `claude login`) are now detected and routed through the Spoof identity
  path. `AnthropicClient` auto-prepends the canonical Claude Code identity
  blocks (billing header + identity prefix) to the system field, making
  OAuth-authenticated requests indistinguishable from real Claude Code
  traffic to the Anthropic Messages API.
- This unblocks real-LLM Tier 1 + specialist execution end-to-end on
  developer credentials. Verified live: a Cournot situation produces
  high-confidence Tier 1 fingerprints (continuous strategy + simultaneous
  + non-cooperative + variable-sum) that the keyword-regex fallback could
  not generate.

### Completion integrity

- `archon completion verify` no longer panics from nested tokio runtimes
  (handlers now async end-to-end).
- `verify --require-claims` flag added for CI strictness.
- TestsPassGate + IngestedGate + AnswerGroundedGate now correctly block
  `Verified` verdict when evidence has `Missing` status.
- `agent_model_trust_scores` automation: trust scores now compute from
  incident/completion history.

### Governed learning

- Phase 5 placeholder `learning_events_pending` is now consumed by
  `archon-learning::outcome_signal::consume_pending`, translating
  completion incidents into canonical `learning_events` rows.
- `archon behaviour {list-events, list-proposals, show, approve, reject,
  apply, rollback, history, generate-proposals}` ŌĆö full governance
  surface.
- Manifests are versioned + immutable; rollback creates a new version
  pointing at the target's content rather than overwriting.
- Policy engine hard rules: HighRisk and Critical changes always require
  approval; PolicyOverride manifest changes are always Critical.

### Document intelligence + retrieval

- Hybrid retrieval (`archon docs search --mode {exact|semantic|hybrid}`).
- `archon docs answer` persists answer-to-chunk provenance edges and prints
  an Answer ID for downstream verification.
- Image OCR + VLM policy gate (default-disabled) wired.
