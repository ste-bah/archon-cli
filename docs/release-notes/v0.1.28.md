# v0.1.28 — ReasoningBank 12-mode wire-up + README accuracy sweep

## What shipped

### A. ReasoningBank now actually has 12 modes

Previously, `ReasoningMode` enum had 4 variants (PatternMatch, CausalInference, Contextual, Hybrid). Twelve mode engine modules existed under `crates/archon-pipeline/src/learning/modes/` (3,051 lines) but `ReasoningBank::reason()` ignored them. Wired up:

- **ReasoningMode enum** extended from 4 to 14 variants: 12 spec modes (Deductive, Inductive, Abductive, Analogical, Adversarial, Counterfactual, Temporal, Constraint, Decomposition, FirstPrinciples, Causal, Contextual) plus 2 meta-modes (PatternMatch, Hybrid).
- **`reason()` dispatch** extended to 14 match arms; 10 new arms call the existing engine modules via `reason_via_engine` helper that adapts `ReasoningOutput` → `ReasoningResponse`.
- **`ModeSelector::select()`** keyword heuristics for all 14 patterns ("what if" → Counterfactual, "why did" → Abductive, etc.).
- **`ReasoningBankConfig`** extended with 13 per-mode weights for Hybrid aggregation.
- **`CausalInference` renamed to `Causal`** with `#[serde(alias = "CausalInference")]` for backward compat with old config files.

### B. Tests

14 new tests in `tests/reasoning_bank_12_modes.rs`:
- One per mode: dispatch verification, non-zero confidence, engine_name in trajectory metadata
- `auto_mode_selection_picks_correct_mode_for_each_query_pattern` (12-row table)
- `hybrid_mode_aggregates_results_from_multiple_sources` (≥3 distinct mode sources)

14 existing tests in `reasoning_bank_tests.rs` still pass after the rename.

### C. README accuracy sweep

Four user-facing drift fixes:

## Background

This was the long-standing "module exists but not wired" failure mode applied to the learning systems. v0.1.23's release notes correctly stated that 4 modes were wired, but the README's headline claims (12 modes) and the ReasoningBank diagram both promised more than was actually delivered. v0.1.28 makes the README claims true.
