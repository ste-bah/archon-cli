# v0.1.47 — Unified PDF Text, OCR, And VLM Image Ingest

v0.1.47 extends the v0.1.46 VLM image pipeline to the document type people
actually ingest most: PDFs. `archon docs ingest` now handles native-text PDFs,
mixed text-plus-image PDFs, and scanned/image-only PDFs through one pipeline.

## What Changed

- Text-layer PDFs still use `pdftotext -layout` and persist normal page/chunk
  provenance.
- Mixed PDFs now run `pdfimages` to extract embedded charts, figures, diagrams,
  and photos. Surviving images get local OCR plus optional VLM descriptions.
- Scanned/image-only PDFs fall back to `pdftoppm` page rendering, then OCR and
  optional VLM descriptions per rendered page.
- PDF image descriptions are stored in `doc_image_descriptions`, chunked into
  normal `doc_chunks`, and searchable through existing exact/semantic/hybrid
  retrieval.
- `archon docs status`, `archon docs inspect`, and `archon docs model-status`
  now expose PDF image extraction diagnostics.

## Policy

New defaults in `.archon/policy.toml` and `scripts/archon-init.sh`:

```toml
[policy.docs.pdf]
extract_embedded_images = true
min_image_dimension = 200
min_image_bytes = 4096
vlm_per_page_image = true
render_text_pdf_pages = false
```

`vlm_per_page_image` does not bypass `[policy.docs.vlm]`. Local/cloud VLM gates
still apply exactly as they do for standalone images.

## Migration Note

Existing v0.1.46 document stores remain valid. To make older PDF corpora
searchable by visual content, re-ingest those PDFs so Archon can extract image
OCR and VLM description chunks.
