## Provider doctor + capability matrix

New CLI: `archon providers doctor [--live]`. New TUI: `/providers doctor [--live]`. Reports:

- Credentials file presence + path
- Anthropic OAuth state (present / missing / expired)
- Codex OAuth state + kill-switch (`ARCHON_CODEX_DISABLED`)
- `ANTHROPIC_API_KEY` env-var shape (`sk-ant-api-*` vs `sk-ant-oat-*` vs proxy)
- `ANTHROPIC_BASE_URL` / proxy state
- Spoof-identity status per provider
- Optional live endpoint reachability (`--live`) via `TcpProviderLivePinger`
- Redacted remediation hints

Token values are NEVER printed. Enforced by unit test `render_provider_doctor_live_uses_pinger_without_printing_tokens`.

`archon providers capabilities` + matrix gating: in v0.1.44, Codex provider was intentionally blocked from agentic surfaces (pipelines, subagents, gametheory). The capability matrix was the source of truth — verified by `provider_capability_matrix_documents_codex_tui_but_not_pipelines`. v0.1.45 replaces that historical limitation with tested Codex agentic parity.

19 providers tests + 6 finalisation tests cover the new surface.

## TUI hang fix (the v0.1.44 outline)

The 2026-05-05 forensic finding: TUI render loop blocked inside `terminal.draw` for 8+ minutes processing a 200 KB DocAnswer chunk dump. 5.96 GB RSS bloat from the unbounded TuiEvent channel filling up while producers kept firing. Two fixes:

1. **Bounded event channel** — `crates/archon-tui/src/event_channel.rs` (NEW, 253 lines). Channel capacity 1024. Producers `.await` when full instead of growing the queue unbounded. Prevents the RSS bloat under any future stuck-render scenario.
2. **Viewport-cached output rendering** — `crates/archon-tui/src/output/render_cache.rs`. Spans are pre-computed on append; per-frame the renderer slices the scroll-window only. Eliminates whole-buffer re-parse per draw on long DocAnswer dumps.

Both fixes verified by `crates/archon-tui/tests/render_coverage.rs` (538 lines, fixture-rich).

Recommendation #1 from the v0.1.44 outline (cap DocAnswer output in the tool itself) is NOT in this release — Codex took the architectural path (#2 + #3) instead. The current DocAnswer behaviour (full chunk dump) is acceptable now that the TUI is no longer hangable by it. If you still want a tool-side cap as a UX improvement, file a separate ticket.
