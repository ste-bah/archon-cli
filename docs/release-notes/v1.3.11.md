# Archon v1.3.11 — Trading Lab

v1.3.11 adds the governed Trading Lab substrate: a research-first, evidence-led
foundation for strategy specification, Pine prototyping, deterministic
backtesting, paper-trading gates, live-readiness checks, and audit/learning
integration.

## Added

- `archon-trading` crate surfaces for strategy knowledge, spec registry,
  Pine-script prototypes, market-data registration, deterministic backtests,
  regime labelling, paper/live terminal models, live enablement, dry-run
  certification, risk policy, postmortems, promotion review, agent policy, and
  audit ledgers.
- TradingView MCP and OpenBB adapter primitives for governed Pine/data
  workflows without embedding provider credentials in strategy records.
- Project-local Trading Lab setup via `scripts/setup-trading-tools.sh`,
  including `tradesdontlie/tradingview-mcp`, an OpenBB virtualenv, `.mcp.json`
  MCP registration, and helper scripts for TradingView CDP and OpenBB API.
- `archon trading` / `/trading` subcommands for external tool status,
  TradingView MCP CLI pass-through, Pine v6 generation/analyze/check, governed
  OpenBB fetches with optional direct OHLCV persistence, StrategySpec
  validation, persistent OHLCV data-lake ingest, native fill and candle
  backtests, custom deterministic strategy-rule
  backtests, paper-order/risk checks, TradingView replay-paper submit, Trading
  Lab workflow spec generation, promotion checks, and live-readiness gates.
- Learning hooks that turn trading research outcomes, postmortems, blocked
  unsafe attempts, and data-quality failures into auditable learning inputs
  without allowing learning to bypass risk policy.
- TUI trading panel model and `archon-tools` trading dispatch primitives wired
  into the first-class `/trading` command surface.
- [Trading Lab reference](../trading-lab.md) covering the current architecture,
  implemented modules, safety posture, and what users can do today.
- [Trading Lab cookbook](../cookbook/trading-lab.md) with a detailed practical
  path from source ingestion and KB creation through strategy specs, Pine
  variants, deterministic backtests, paper gates, postmortems, promotion
  review, and live-readiness checks.

## Changed

- README and docs index now include Trading Lab reference and cookbook entry
  points.
- Hybrid docs retrieval now sanitizes user questions before Cozo FTS and skips
  semantic vector warm-up when high-confidence exact evidence is already
  available, including specific definition-style KB questions with one clear
  lexical hit. Minor typos in definition-style questions get a narrow relaxed
  exact retry before semantic fallback. Set
  `ARCHON_DOCS_HYBRID_ALWAYS_SEMANTIC=1` to force the old always-semantic
  hybrid path for diagnostics.
- Agent-callable `DocSearch` and `DocAnswer` now use the in-process docs
  runtime instead of spawning a fresh `archon docs ...` child process, avoiding
  cold-start stalls in TUI/web chat retrieval.
- TUI splash snapshots updated for v1.3.11.
- Root docs ignore rules now explicitly allow `docs/trading-lab.md` so the new
  reference page is tracked with the rest of the user-facing docs.

## Safety Notes

- Live broker trading remains disabled by default.
- TradingView alerts are treated as non-authoritative order intents, not orders.
- Strategy Tester output is auxiliary evidence only; promotion still requires
  Archon backtest evidence with data snapshots, config hashes, costs, and
  robustness checks.
- OHLCV data is stored as versioned market data under
  `.archon/trading-lab/data`, separate from KB/document chunks. Strategy specs
  reference datasets by `dataset_id` and `version`.
- Learning may record lessons and propose process improvements, but it may not
  raise live limits, bypass maker-checker, emit orders, or promote a strategy
  without deterministic evidence.
