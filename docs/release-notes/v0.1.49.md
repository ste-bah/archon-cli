# v0.1.49 — TUI Drain-Stall False Positive Fix

v0.1.49 stops long document ingests from flooding session logs with false TUI
drain-stall warnings.

The stall detector now checks whether TUI events are actually pending before it
warns. Idle channels are treated as idle, not stuck. Long-running slash
workloads such as `/docs ingest <path>` also get a wider drain threshold while
their mirrored CLI task is running.

Repeated warnings are rate-limited to one per minute per active stall episode,
or sooner if the pending queue grows. Real stalls still warn when queued events
pile up and the render loop does not drain them.
