# v0.1.41 — pin-count regression fixes + docs sweep follow-through

Hotfix release for the pin-count test regressions introduced by Codex's v0.1.40 work that escaped CI on the merge commit. Plus a docs/visibility consolidation that landed on main with `[skip ci]` between v0.1.40 and v0.1.41.

## Test fixes (the actual bug)

Four pinned-count tests were stale relative to the v0.1.40 surface and silently failed on the post-merge CI matrix:

- `command::providers::tests::execute_lists_both_section_headers` — pin said "NATIVE (5)" / "OPENAI-COMPAT (31)"; actual is now NATIVE (6) because Codex added `openai-codex` as a native provider in `crates/archon-llm/src/providers/native_registry.rs:148`. Bumped to (6 + 31).
- `command::providers::tests::execute_emits_total_count_line` — pin said "36 total: 5 native + 31 openai-compat". Bumped to "37 total: 6 native + 31 openai-compat".
- `command::providers::tests::execute_total_row_count_matches_registry_size` — pin said `expected exactly 36 provider rows`. Bumped to 37.
- `archon-core::env_vars_tests::known_vars_list_has_correct_count` — pin said `KNOWN_ARCHON_VARS.len() == 23`. Codex added 12 `ARCHON_CODEX_*` env vars and 2 debug entries in `crates/archon-core/src/env_vars.rs`; new count 37 (4 auth + 12 codex + 4 model + 4 feature + 6 debug + 6 paths + 1 telemetry).

Three TUI splash tests were stale relative to the v0.1.40 splash redesign:

- `splash_has_logo` — asserted `text.contains("ARCHON")`. The new compact ASCII figlet doesn't contain the literal word "ARCHON" — it spells it in glyphs. Updated to assert the tagline `evidence + agents` (a stable v0.1.40+ literal in `ASCII_FALLBACK`).
- `draw_splash_shows_tips` — asserted `contains("/model")`. v0.1.40 refreshed the tips block away from `/model to switch models` toward `/auth status`, `/agents`, `/docs ingest`, `/gametheory run`. Updated to assert at least one of `/auth status` or `/docs ingest`.
- `splash_image_area_height_is_12` — counted halfblock U+2580 glyph rows. v0.1.40 hard-coded `use_ascii_fallback = true` in `crates/archon-tui/src/splash.rs:309`, so the halfblock renderer is unused at runtime. Updated to count any non-blank/non-border glyph in the 12-row image area.

The pinned constants themselves (registry sizes + env-var list contents) were CORRECT in v0.1.40; only the test pins lagged behind. Same flavor of mistake as the `EXPECTED_PRIMARY_COUNT 76 → 78` fix that landed in v0.1.40.
