# v1.2.3 - Browser Web Workbench

v1.2.3 adds the browser web workbench: a local-first inspection and
operations surface for Archon state that previously required a mix of
TUI views, slash commands, and shell commands.

## Added

- Embedded React/Vite web workbench served by `archon web`, with the
  production bundle shipped from `web/dist` inside the Rust binary.
- Browser tabs for overview, chat, corpus, memory and learning, world
  model and reasoning-quality data, pipelines, metrics, settings, and
  the evidence graph.
- Read-only inspection APIs for web status, effective config and
  policy, corpus summaries, learning rows, world-model state, pipeline
  state, provider telemetry, and settings.
- Typed web action envelope with dry-run, confirmation, policy, and
  audit metadata for mutating actions. Evaluation uses the effective
  `[policy.web]` gates plus subsystem gates such as
  `policy.world_model.allow_behavior_changes`.
- Live event manager API with cursor replay, cursor-expired recovery,
  and backpressure compaction events.
- Attachment/upload metadata model backed by `[policy.web]` upload gates
  for the browser chat surface.
- Web chat send/attach controls now perform real client actions: send records
  a local web-session message and attach validates selected files through the
  upload-intent policy endpoint.
- Evidence graph counts derived from actual local corpus files and scanned
  text markers instead of fixed synthetic multipliers.
- Generated TypeScript DTO gate from Rust API structs, plus strict
  TypeScript, Playwright screenshot coverage, and Vite development mode.

## Changed

- README and installation docs now describe blank-project setup and how
  to launch the browser workbench from an initialized project root.
- File-size guard now covers hand-written TypeScript, TSX, and CSS
  alongside Rust source files.
- TUI splash snapshots and workspace package metadata are aligned to
  `v1.2.3`.
- Default policy templates now include the `[policy.web]` browser workbench
  gates, all disabled by default.
- Web metrics now report resolved runtime storage paths, including the
  configured memory database path and embedded web assets, rather than guessed
  `~/.archon/memory` or source-tree `web/dist` locations.

## Notes

- The web workbench is local-first. Binding to `127.0.0.1` is the safe
  default; non-loopback bindings require the existing web auth posture.
- Normal users do not need Node.js. Node 22+ is only required when
  developing or rebuilding the web frontend.
