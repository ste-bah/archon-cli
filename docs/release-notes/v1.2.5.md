# v1.2.5 - Provider Tier Alias Resolution And Anthropic Message Sanitizer

v1.2.5 fixes three production bugs surfaced in a live Anthropic session:

- Subagents declared with `model: opus|sonnet|haiku` in their frontmatter
  hit HTTP 404 `model: opus` because the provider's `stream()` never
  resolved tier aliases. The pipeline LLM adapter happened to resolve at
  its call site, masking the gap for pipelines but breaking every
  subagent. v1.2.5 moves alias resolution into the provider boundary
  itself so every direct caller is covered.
- Compaction was producing `role: "system"` boundary messages that
  Anthropic rejects with `messages: Unexpected role "system"`. The fix
  switches the boundary to `role: "user"` with a marker prefix and adds
  a sanitizer at the Anthropic request boundary so any future code path
  that leaks `role: "system"` into `messages` is normalized before send.
- Compaction could split a `tool_use` block from its matching
  `tool_result`, producing the `tool_result block must have a
  corresponding tool_use block in the previous message` error. v1.2.5
  makes the compaction split-point pair-safe so the boundary slides
  until both `tool_use` and `tool_result` fall on the same side.

Also folded in: CI workflow apt-get hardening so Azure Ubuntu mirror
hangs no longer consume 30-50 minute job slots in silence.

## Fixed

- **Provider tier alias resolution at the provider boundary.** All
  seven providers (`anthropic`, `codex`, `openai`, `local`,
  `openai_compat`, `bedrock`, `vertex`) now call
  `self.resolve_request_model(&mut request)` at the top of `stream()`.
  Trait method `LlmProvider::resolve_request_model` resolves through
  the per-provider `resolve_alias` impl first and falls back to the
  first advertised model when the request specifies an empty model or
  any tier alias (`opus`, `sonnet`, `haiku`, `default`, `mini`,
  `codex`). No more silent pass-through of literal alias strings to
  the upstream API.
- **Anthropic request sanitizer for `role: "system"` and orphan
  `tool_result`.** `AnthropicProvider::stream()` runs
  `sanitize_anthropic_messages(request.messages)` before dispatching.
  Non-`user`/`assistant` roles are normalized to `user`; a user
  message containing a `tool_result` whose `tool_use_id` does not
  match a `tool_use.id` in the immediately preceding assistant message
  is demoted to plain text. Last line of defense — the real fixes are
  the pair-safe compaction and the batched tool-result builder, but
  the sanitizer turns any future bug into a soft demotion instead of
  an Anthropic 400.
- **Pair-safe compaction boundaries.**
  `archon_context::compact::safe_tail_start()` walks the requested
  split forward; if the message at the split has `tool_result` blocks
  whose ids match `tool_use` blocks in the assistant message at
  `split - 1`, the split backtracks one message so both halves of the
  pair stay together. Orphaned `tool_result` messages at the start of
  the retained tail are compacted away instead of leaking through.
- **Compaction boundary message uses `role: "user"`, not
  `"system"`.** `archon_context::boundary::CompactBoundary::to_message`
  now emits a `user` role. Combined with the new sanitizer this means
  even an out-of-band code path that emits the boundary into a
  message vec will not poison the next Anthropic request.
- **Parent tool results batch into one user message.**
  `ConversationState::add_tool_result` now appends to the last user
  message if it is already an all-`tool_result` block, otherwise
  creates a new one. Matches the safer subagent shape and keeps every
  `tool_use` paired with all its `tool_result` siblings in a single
  contiguous user message.

## Changed

- **CI apt-get hardening.** All twelve `sudo apt-get update`
  invocations across `.github/workflows/release.yml`, `ci.yml`, and
  `tui-observability.yml` are wrapped with
  `timeout 120 ... -o Acquire::Retries=3 -o Acquire::http::Timeout=30`.
  Azure Ubuntu mirror hangs now fail in two minutes instead of
  consuming 30-50 minute timeout budgets. Triggered by the 2026-05-13
  workflow runs where four jobs (`clippy advisory`,
  `cognitive complexity`, `observability coverage`, `.send().await
  agent_event_tx`) stalled silently mid-apt for between five and
  fifty-two minutes.
- Workspace and web package metadata aligned to `1.2.5`.
- TUI splash snapshots bumped to `Archon v1.2.5`.

## Notes

- File-size guard still passes — no source file over 500 lines.
- No public API changes. `LlmProvider::resolve_request_model` is a
  trait method with a default implementation (resolve_alias → first
  advertised model), so existing provider impls keep compiling without
  changes. New providers only need to override `resolve_alias` when
  they have a non-trivial alias map.
- The Anthropic sanitizer is a safety net, not a substitute for
  correct message construction. Existing code paths that produced
  orphan `tool_result` blocks or `role: "system"` messages have been
  fixed at the source — the sanitizer protects against future
  regressions in those paths.
