# v0.1.45 — Codex agentic provider parity and production polish

v0.1.45 closes the Provider Parity finalisation slice: Codex OAuth is no longer
documented or capability-gated as chat/TUI-only. When `[llm].provider =
"openai-codex"` is selected, Archon routes the same agentic surfaces through
Codex that it routes through Anthropic, unless a provider limitation is proven
and reported before execution.

## What changed

- `openai-codex` capability rows now mark chat, TUI, streaming, tool use,
  subagents, `/btw`, team runs, coding/research pipelines, and gametheory as
  supported.
- Archon can now read the official Codex CLI's `~/.codex/auth.json` as a
  read-only fallback when `~/.archon/.credentials.json` has no
  `openaiCodexOauth` entry.
- Codex-named subagent tests now cover both tool-result continuation and two
  concurrent subagents reaching the first provider turn without serializing.
- Completion integrity keeps provider-neutral source-of-truth rows: Codex-shaped
  `agent_key` and `model` metadata persist to completion claims, run contexts,
  incidents, and trust scores.
- TUI activity rows carry provider/model/cost metadata where known, so parent,
  tool, subagent, and pipeline activity can be inspected without guessing which
  backend is active.
- Gametheory slash inspect commands reopen Cozo per read, so same-session
  `classify-only -> inspect-fingerprint -> inspect-routing -> show` reads see
  rows written by background slash tasks.
- TUI event-drain forensics records the last drained event variant and
  per-variant counts, making Ctrl-C/bootstrap stalls diagnosable rather than
  anonymous.

## Docs and operations

- README, Codex auth, identity spoofing, provider capabilities, CLI/slash
  references, env vars, and cookbook workflows now explain Anthropic OAuth,
  Anthropic API keys, Anthropic-compatible proxies, and Codex OAuth as separate
  provider choices.
- Provider smoke and Codex smoke runbooks explicitly forbid scheduled live
  provider cron jobs. Live checks remain manual because they can spend quota.
- Real-world examples now show how to drive document intelligence, KB,
  gametheory, coding, research, completion integrity, meaning, and
  constellation flows from both shell and TUI.
