# Archon v1.3.3 — Video Evidence Ingest

## New Features

- Added `archon video ingest` for local files, direct URLs, transcript-only URLs,
  and policy-gated acquisition flows.
- Added the `archon-video` crate with video source metadata, transcript segments,
  frame descriptions, timestamp references, and frame artifact persistence.
- Added transcript parsing and export for VTT, SRT, TTML, JSON, and timestamped
  text; `archon video transcript <video-id> --format txt|srt|vtt` now reads the
  persisted transcript track.
- Added local ASR interfaces, VAD timestamp stabilization, optional diarization
  hooks, GPU backend probing, and managed model-cache resolution.
- Added frame extraction through `ffmpeg`, perceptual hash dedupe, OCR chunks,
  and VLM frame descriptions using the dedicated video-frame prompt.
- Added `video@MM:SS` answer citations via `video_chunk_timeref` without making
  `archon-docs` depend on `archon-video`.
- Added optional per-video LLM summaries, default off and fully policy-gated.
- Added `/video` as a TUI slash mirror for the `archon video` CLI namespace.

## Policy Changes

- Added default-deny `[policy.video]` gates for source acquisition, caption
  capture, ASR/VLM cloud flow, download limits, frame extraction, ASR settings,
  and summary generation.
- External downloader and browser-acquisition paths are explicit opt-ins and do
  not attempt proxy rotation, CAPTCHA bypass, fingerprint spoofing, or other
  platform-evasion behaviour.

## Evidence Engine

- Video transcript, OCR, VLM, and summary evidence is stored as ordinary
  `doc_chunks`, so existing document search and `archon kb process` consume
  video evidence automatically.
- `archon video inspect` now reports persisted tracks, segments, frame rows,
  time references, document chunks, and provenance counts.

## Documentation

- Added [Video Evidence Ingest](../video.md).
- Updated document-intelligence, policy, README, and slash/CLI examples for the
  video evidence workflow.
