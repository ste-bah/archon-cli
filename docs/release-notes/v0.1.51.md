# v0.1.51 - Corrections Feed Behaviour Proposals

v0.1.51 closes the loop between user corrections and governed learning.

Detected user corrections now emit `UserCorrected` learning events with the
correction type, a short input excerpt, and the associated behavioural rule id.
When three or more corrections cluster on the same rule within seven days,
the proposal engine emits a `BehaviouralRuleAdjustment` proposal for operator
review.

Those proposals appear under `archon behaviour list --pending` and can be
inspected, approved, or denied through the existing governed-learning flow.
No correction is applied automatically.
