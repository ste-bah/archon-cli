# v0.1.48 — OpenAI-Compatible VLM And Gemini Retry Hardening

v0.1.48 adds a fourth document VLM provider and makes Gemini free-tier bursts
survivable during large image/PDF ingests.

## Highlights

- New `openai-compat` VLM provider for LM Studio, vLLM, llama.cpp server, and
  `api.openai.com` vision-capable chat-completions models.
- Local OpenAI-compatible servers send no `Authorization` header when the
  configured API-key env var is unset; cloud endpoints use
  `Authorization: Bearer ...` from the configured env var.
- `archon docs model-status` and `archon providers doctor --live` now diagnose
  the configured OpenAI-compatible VLM endpoint through `/models`.
- Gemini VLM retry now allows five attempts with exponential backoff and lowers
  the default free-tier rate limit from 15 RPM to 12 RPM.

## Policy

```toml
[policy.docs.vlm]
enabled = true
mode = "local"
provider = "openai-compat"

[policy.docs.vlm.openai_compat]
endpoint = "http://localhost:1234/v1"
model = "google/gemma-3-12b-it"
api_key_env = "OPENAI_API_KEY"
timeout_secs = 120
max_tokens = 1024
temperature = 0.2
```

For OpenAI's hosted API, use `mode = "cloud"`, set
`policy.docs.vlm.allow_cloud = true`, set `policy.network.allow_cloud_vlm =
true`, and point `endpoint` at `https://api.openai.com/v1`.
