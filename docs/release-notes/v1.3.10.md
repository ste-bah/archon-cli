# Archon v1.3.10 — Dynamic Workflows

v1.3.10 introduces provider-neutral dynamic workflows: generated, durable
multi-stage work plans that can be planned, run, paused, resumed, inspected,
and saved as templates without binding the workflow schema to one LLM vendor.

## Added

- `archon workflow plan|run|status|resume|pause|cancel|restart-agent|force-accept|save|list`.
- Workflow CLI `--live` mode for real provider-backed `plan`, `run`, and
  `resume`, while default shell execution remains deterministic smoke mode for
  cheap orchestration tests.
- Workflow spec/template reuse with `archon workflow run --spec-file <path>`,
  `archon workflow plan --spec-file <path>`, and
  `archon workflow run --from-template <name>`.
- `/workflow` TUI slash command parity through the command registry, including
  live TUI planning/execution through the active LLM adapter for `plan`, `run`,
  and `resume`.
- `archon-workflow` crate with validated YAML workflow specs, dependency-cycle
  checks, provider-tier intent, deterministic reducers, quality gates, durable
  manifests, event logs, and artifact hashes.
- Sanitized workflow event records that strip provider-private reasoning fields
  before persistence or UI/web projection.
- Dynamic workflow Agent Activity rows showing run id, stage/agent, status,
  provider-tier detail, and resolved model alias.
- Web Workbench Workflows page backed by summary, detail, incremental event,
  SSE stream, and policy-gated control APIs.
- Workflow learning ledgers under `.archon/workflows/<run-id>/learning/` with
  accepted-only durable-memory records and direct adapter records for SONA,
  RLM, Reflexion, ReasoningBank, JEPA, and world-model consumers.
- Audited workflow `force-accept` for manually reviewed failed stages; forced
  stages remain audit-visible and do not become clean durable memory.
- Reusable workflow template storage under `.archon/workflow-templates`.
- Monotonic event sequencing across restart/resume cycles.
- Dynamic workflow architecture and cookbook documentation.

## Changed

- Slash command count is now 83.
- Pipeline documentation now distinguishes audited fixed pipelines from
  dynamic generated workflows.
