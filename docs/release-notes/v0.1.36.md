# v0.1.36 ‚Äî Trajectory Embeddings + Persistence

The GNN auto-trainer (v0.1.26) has been training on zero-vector placeholder embeddings
since launch. This release wires the full pipeline: schema, migration, production write
path, embedding computation, and read path.

## What's new

- **`embedding` column on `trajectories`** ‚Äî Schema gains a 13th column (`embedding: [Float]`)
  between `context` and `quality`. Existing databases auto-migrate via a new migration
  runner with version sentinel tracking.
- **Production write path** ‚Äî `SonaEngine` now optionally persists trajectories to
  CozoDB via `trajectory_store::store_trajectory`. The in-memory HashMap remains for
  in-session fast access; CozoDB handles cross-session persistence the auto-trainer reads.
- **Embedding pipeline** ‚Äî Trajectories are embedded at first non-zero quality feedback
  using the configured `EmbeddingProvider`. Embeddings are padded/truncated to
  `[learning.gnn] input_dim` (default 1536).
- **Read path with dimensional filtering** ‚Äî `query_trajectories_for_training` now extracts
  real embeddings and skips rows with empty or wrong-dimension embeddings. The
  placeholder `vec![0.0f32; 1536]` is gone.

## Migration

Schema migration from v1 (12 columns, no embedding) to v2 (13 columns) runs automatically
on first launch against an existing learning database:

1. `learning_schema_version` sentinel tracks current version
2. v1 rows are migrated row-by-row in Rust with empty `embedding = []` sentinels
3. Empty-embedding rows are filtered at query time (they need re-embedding via feedback)
4. Fresh databases install v2 directly ‚Äî no migration needed

No data loss. Migration is idempotent (safe to re-run).

## Dimension caveat for local embedding providers

The local fastembed provider (`BGEBaseENV15Q`) produces **768-dimensional** vectors.
When the GNN expects 1536-dim input, vectors are zero-padded (50% zero suffix),
which degrades model quality proportionally.

For full-quality training, configure:
```toml
[memory]
embedding_provider = "openai"
```
and set `OPENAI_API_KEY` in the environment.
