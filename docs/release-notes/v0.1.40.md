# v0.1.40 - OAuth docs and TUI agent activity

## Highlights

- Documents both supported subscription-auth paths: Anthropic/Claude OAuth through Claude Code identity spoofing, and OpenAI Codex OAuth through the `openai-codex` provider.
- Bumps the workspace version to `0.1.40`.
- Allows full interactive TUI sessions to use Codex when `[llm].provider = "openai-codex"`, without requiring Anthropic auth to exist first.
- Refreshes the TUI splash screen with a crisp text logo, auth-aware tips, and a less empty first-run layout.
- Adds a compact Agent Activity rail that shows the parent turn, active tools, and subagent tool launches from the existing TUI event stream.

## Operator notes

Use `archon auth login --provider anthropic` for Claude-backed agents and pipelines. Use `archon auth login --provider openai-codex` plus `[llm].provider = "openai-codex"` for Codex-backed TUI sessions, or `archon chat --provider openai-codex` for one-shot chat. `archon auth status` reports both paths without printing secret tokens.

## Verification

This release keeps the OAuth spoof behavior already covered by `archon-llm` and `archon-sdk` tests, and adds TUI state/render coverage for the activity rail.
