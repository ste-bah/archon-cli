# v0.1.24 — GNN forward pass parity with TypeScript (PR #12)

PR 1 of the GNN port saga. Faithful round-trip 3-layer GNN matching root archon's TypeScript reference implementation.

## Architecture

`1536 → 1024 → 1280 → 1536` round-trip with:
- 12 attention heads
- He initialization for ReLU activations
- Residual connections (active where dims match)
- Layer normalization
- LRU + TTL cache

Round-trip preserves embedding dimensionality so enhanced vectors can replace originals in the same vector store.

## What shipped

- Ported `gnn-enhancer.ts`, `gnn-math.ts`, `gnn-cache.ts` to Rust (`archon-pipeline` crate)
- **xoshiro128\*\* PRNG** with non-standard output function matching JS `Math.imul` exactly (so weight initialization matches the TS reference byte-for-byte)
- **FNV-1a stride-4 cache key** byte-matching TS Math.imul behaviour
- **Scaled dot-product graph attention** with adjacency-weighted softmax
- **Numerically stable softmax** with uniform fallback on zero-sum
- Pure `Vec<Vec<f32>>` arithmetic — no new ML crate dependencies

## Acceptance gate

Cosine similarity ≥ 0.999 vs 10 TS reference fixtures. Fixtures generated once via `npx ts-node scripts/generate-gnn-fixtures.cjs --seed 12345 ...` and committed to the repo at `tests/fixtures/gnn_expected.json`. Tests read the JSON only — no Node, no ts-node, no network during `cargo test`.
