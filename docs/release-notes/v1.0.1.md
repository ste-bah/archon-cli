# v1.0.1 - Hybrid Retrospective Analysis

v1.0.1 is a patch release on top of the v1.0.0 audited pipeline runtime and
self-calibration baseline.

`archon self retrospective <session-id>` now supports
`--analyzer hybrid|heuristic|llm`. The default `hybrid` mode keeps the
deterministic local extractor for source-tree mistakes, tool or agent failures,
and verification habits, then adds a provider-neutral LLM pass through the
active configured `LlmProvider`.

That keeps Anthropic OAuth, Codex OAuth, and compatible providers on the same
retrospective analysis path. LLM candidates are only accepted after real
evidence-event-id, confidence, duplicate, and secret-shape checks. If the
provider or credentials are unavailable, Archon records an analyzer note in the
retrospective report and falls back to deterministic candidates.

Docs and command-surface metadata have been updated to describe hybrid
retrospectives without changing the slash-command count or audited pipeline
runtime surfaces.

The repo config and policy templates now explicitly show the `openai-codex`
provider route used by hybrid retrospectives and the complete
OpenAI-compatible VLM policy example.
