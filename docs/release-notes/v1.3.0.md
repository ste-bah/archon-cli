# v1.3.0

v1.3.0 removes a model-facing subagent turn cap footgun and tightens the TUI
signals added during the auto-compaction work.

## Subagents

- `max_turns` is no longer advertised in the `Agent` or `TaskCreate` JSON
  schemas. Parent models can no longer bound long-running subagents by adding a
  small turn count such as 20 or 45 to their tool input.
- The internal `SubagentRequest.max_turns` field, `DEFAULT_MAX_TURNS = 100_000`,
  `MAX_TURNS_HARD_CAP = 100_000`, custom-agent `meta.json` overrides, built-in
  agent overrides, and the CLI `--max-turns` flag all remain available for
  operator-controlled caps.
- The validators still defensively accept `max_turns` when supplied by older or
  non-schema callers, but now log a warning so production logs reveal any model
  still emitting the removed field.

## TUI

- Foreground activity stream metadata now preserves a subagent event's own
  provider/model instead of overwriting it with the parent session model. A
  `sherlock-holmes` actor configured as `opus` now displays as `opus` while the
  parent can still show `claude-sonnet-4-6`.
- The bottom status bar now shows the resolved context window before the first
  token-usage event, so startup sessions display `ctx ... 0k/<window>`.

## Web workbench

- Web chat now returns the live assistant response instead of falling back to a
  "recorded" policy message when streamed output is empty.
- Attachments are encoded at submit time and passed into the document ingestion
  path so uploaded PDFs and text files can be used as model context.
- Read-only document evidence tools (`DocAnswer`, `DocSearch`, `DocInspect`,
  and related inspection tools) are safe-listed in default/auto/plan permission
  modes. `DocIngest` remains gated.
- Memory filter chips now update a visible row preview, and metrics ledgers
  under existing stores report as `quiet` instead of `missing`.

## Tests

- Added schema absence tests for both `Agent` and `TaskCreate`.
- Added a default-path regression proving model input that omits `max_turns`
  gets `DEFAULT_MAX_TURNS`.
- Added activity metadata and startup context-window status regressions.
- Added web chat, metrics ledger, and read-only document permission regressions.
