# Archon v1.3.7 — Autonomous Governed Learning

v1.3.7 makes Archon's governed-learning loop capable of policy-gated self-application while keeping dangerous change families behind hard stops.

## Highlights

- Adds `archon learning tick` to generate, deduplicate, evaluate, and optionally auto-apply behaviour proposals from recorded learning events.
- Adds `[policy.learning]` autonomous controls:
  - `autonomous_apply`
  - `autonomous_max_risk`
  - `autonomous_min_evidence`
  - `autonomous_max_recent_incidents`
- Keeps `PolicyOverride` proposals human-gated even when autonomous learning is enabled.
- Wires pipeline correction clusters into the same governed-learning evaluator, so pipeline self-improvement follows one policy model.
- Resolves pipeline subagent tier aliases through the active provider before emitting TUI activity, so Codex, Anthropic, DeepSeek, local, and compatible providers display concrete model IDs instead of stale tier names.

## Notes

Autonomous learning is disabled by default. To enable it, set `[policy.learning].autonomous_apply = true` and keep the risk/evidence gates conservative until the project has enough correction history.
