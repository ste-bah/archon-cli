# Archon v1.3.4 — DeepSeek Provider Parity

## Provider Runtime

- Fixed TUI/session model selection so non-Anthropic providers no longer inherit
  a stale Claude model from `[api].default_model`.
- Added Anthropic-format base URL normalization so provider base URLs such as
  `https://api.deepseek.com/anthropic` are expanded to `/v1/messages`.
- Added `ANTHROPIC_MODEL` support for Anthropic-format sessions, matching
  DeepSeek's Claude Code setup style.
- Updated the OpenAI-compatible DeepSeek registry default to `deepseek-v4-flash`.

## TUI

- `/model` now accepts generic provider model IDs in non-Claude/non-Codex
  sessions, including DeepSeek model IDs such as `deepseek-v4-pro[1m]`.

## Documentation

- Documented the recommended DeepSeek Anthropic API setup for full TUI,
  subagent, and pipeline compatibility.
