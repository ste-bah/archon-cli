# v1.2.4 - Metrics Provider Event Tail And Ledger Activity Panel

v1.2.4 is a follow-up patch on top of v1.2.3's browser web workbench. It
fixes a real bug in the Metrics tab — the Provider Event Tail was blank
because the web metrics API was missing the session-store learning DB —
and renames the misleading "Queue depth" panel to "Ledger activity" so
operators are not led to think those rows are runtime queues.

Also folded in: the earlier `fix: wire web chat controls and runtime
paths` work that closed the gap between the chat surface in the web
workbench and the runtime path resolution used by every other web tab.

## Fixed

- Web Metrics Provider Event Tail was blank for live sessions. The web
  metrics API at `crates/archon-sdk/src/web/metrics.rs` only checked a
  short candidate list for `learning.db` and stopped at the first
  matching path, so it never reached the session-store learning DB at
  `~/.archon/sessions/learning.db` where the provider runtime actually
  writes events. v1.2.4 scans **all** relevant `learning.db` candidates
  (workspace `.archon/learning.db`, `~/.archon/learning.db`, and the
  session-store DB), merges provider events across them, dedupes by
  `event_id`, and sorts newest first.
- Missing ledger files now report `missing` instead of the misleading
  `quiet`, so operators can tell "no events yet" apart from "ledger
  file does not exist on disk".
- Metrics page renamed `Queue depth` to `Ledger activity` because the
  numbers are append-only ledger row counts, not runtime queue depths.
  Snapshot fixtures updated.
- Web chat controls and runtime paths are now wired through the same
  `WebRuntimePaths` resolution as the rest of the web API, so the chat
  surface no longer special-cases its own path layout. Regression tests
  added.

## Added

- Regression tests in `crates/archon-sdk/src/web/metrics/tests.rs`
  cover the provider tail path issue and the missing-ledger state.
- E2E mocks and Metrics snapshots updated to reflect the new
  `Ledger activity` label and the multi-source provider event tail.

## Changed

- Workspace and web package metadata aligned to `1.2.4`.
- TUI splash snapshots bumped to `Archon v1.2.4`.

## Notes

- File-size guard still passes — `metrics.rs` is 496 lines, under the
  500-line ceiling.
- No public API changes. The `MetricValue` shape returned by the web
  Metrics endpoint is unchanged; only the labels and the path-scanning
  strategy are different.
