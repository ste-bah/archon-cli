# Archon v1.3.8 — Cognitive Executive Loop

v1.3.8 adds a policy-governed cognitive controller that coordinates memory,
reasoning quality, SelfModel, JEPA/world-model scoring, verification contracts,
reflection, and autonomous maintenance without storing hidden chain-of-thought.

## Added

- New `archon-cognitive` inspection surface for safe executive-state summaries.
- New `archon cognitive` CLI namespace: `status`, `tick`, `daemon`, `inspect`, `self-model`, and `reflections`.
- New `/cognitive` TUI slash surface and Executive State pane.
- New web `/api/cognitive/summary` endpoint and Cognitive dashboard page.
- Integration smoke tests for greeting, code-change, CI-debug, and promoted JEPA/world-model flows.
- Project config templates for `[learning.cognitive]`.
- Policy templates for `[policy.cognitive]`.
- Rust-based cognitive daemon runner with lock/state files and an extensible job trait.

## Changed

- CI-debug candidate ranking now prefers fetching current CI/log evidence over stale document/memory answers.
- Command-surface matrix now records `archon cognitive ...` to `/cognitive` parity.
- TUI splash snapshots updated for v1.3.8.

## Safety

- Cognitive decisions store compact summaries, hashes, candidate ids, policy verdicts, and verification contracts.
- Reflections store safe lessons only.
- Raw turn text remains disabled by default.
- Autonomous low-risk apply remains policy-gated and off by default in templates.
- Background daemon startup requires both `[learning.cognitive.daemon].enabled = true`
  and `[policy.cognitive].allow_background_daemon = true`.
