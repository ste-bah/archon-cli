# v0.1.46 — Multi-provider VLM image descriptions

v0.1.46 turns image ingest from OCR-only into OCR plus visual understanding. Images can now be described by Ollama, Gemini, or Anthropic VLM providers, with descriptions persisted as first-class document evidence.

## Highlights

- Added Ollama VLM support with `gemma4:e4b` as the local default.
- Added Gemini VLM support using `GOOGLE_API_KEY` or `archon auth login --provider google`.
- Added Anthropic VLM support through the existing Anthropic API key/OAuth spoofing stack.
- Added `doc_image_descriptions` and searchable description chunks tied to source image provenance.
- Extended `archon docs model-status` and `archon providers doctor --live` with VLM diagnostics.
- Updated generated `config.toml`, workspace `.archon/policy.toml`, and install/init policy defaults with all VLM provider settings.

## Operator Notes

VLM remains off by default. Local Ollama requires:

```toml
[policy.docs.vlm]
enabled = true
mode = "local"
provider = "ollama"

[policy.workers]
vlm = "allow-local"
```

Cloud providers require `policy.workers.vlm = "allow-cloud"`, `policy.docs.vlm.allow_cloud = true`, and `policy.network.allow_cloud_vlm = true`.

See [`docs/integrations/vlm.md`](../integrations/vlm.md) for examples and provider setup.
