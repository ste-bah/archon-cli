# v0.1.25 — GNN training infrastructure (PR #13)

PR 2 of the GNN port saga. Adds Adam optimizer, backpropagation, contrastive loss, and EWC regularization.

## What shipped

- **Adam optimizer** with bias correction, weight decay, NaN/Inf guards, MIN_V_HAT stability floor, and reset()
- **`softmax_backward`**: `sigma_i * (dL/dsigma_i - dot(sigma, dL/dsigma))` matching TS reference
- **`activation_backward` fix**: ReLU/LeakyRelu gate on `pre_activation`, Tanh/Sigmoid formula on `true_post_activation`
- **Residual gradient splitting**: upstream gradient flows to both main path and skip connection
- **`LayerActivationCache`**: added `true_post_activation` field (pre-residual+norm) for correct Tanh/Sigmoid backward
- **Contrastive triplet loss** with hard triplet mining (parity tested against TS reference)
- **EWC (Elastic Weight Consolidation)** regularizer with Fisher information matrix

## Tests

31 new unit tests: 10 optimizer + 11 backprop + 10 loss. Plus 4 integration acceptance tests:
- `gnn_persistence` (3 tests) — CozoDB WeightStore roundtrip, Adam state, fresh store
- `gnn_gradient_correctness` (2 tests) — numerical gradient (f64 tol=2e-3)
- `gnn_training_reduces_loss` (1 test, 3-property acceptance gate) — loss reduction ≥20%, no NaN/Inf, no overfit divergence
- `gnn_rollback` (2 tests) — NaN guard keeps weights finite under lr=1e10

416 lib tests + 4 GNN parity tests pass (0 failures).

## Bugs found and fixed during implementation

1. **Trainer cache staleness** — `forward_all` returned cached embeddings from epoch 0 for all subsequent epochs, making loss appear frozen
2. **CozoDB stored-relation queries** used `{}` named syntax which can't project key columns; migrated to `[]` positional
3. **CozoDB `has_nan` parameter** used `Str "true"` instead of `Bool true`, causing silent query failure
