# v1.3.2

v1.3.2 finalizes the JEPA eval pipeline release path and closes the doc drift
found while preparing the v1.3.1 world-model work for merge.

## JEPA Eval Pipeline

- `archon world eval-jepa` now has an explicit quick/full contract. The default
  quick mode runs Tier-0 metadata gates first and can skip the FastEmbed
  baseline when a candidate cannot promote anyway.
- `archon world eval-jepa <candidate> --full` is the promotion-grade path. It
  records corpus/config fingerprints, eval schema version, baseline comparison,
  backend execution gates, tensor safety, and checkpoint-size evidence for
  `promote-jepa`.
- `promote-jepa` refuses stale, quick-only, schema-mismatched, corpus-mismatched,
  config-mismatched, missing-baseline, and failing eval reports instead of
  trusting a candidate by id alone.
- New eval-run inspection commands expose persisted run state:
  `eval-jepa-status`, `eval-jepa-runs`, and `eval-jepa-cancel`.

## Operator Honesty

- `--background` now returns a clear deferral error until the background worker
  entry point is wired.
- `--backend` and `--no-cache` are parsed but emit visible warnings when they do
  not yet alter the foreground full-eval path.
- Config and policy docs now mark background eval gates as reserved rather than
  implying the worker path is complete.

## Documentation And Maintenance

- Updated the CLI reference, world-model architecture guide, and JEPA cookbook
  to distinguish quick feedback from full promotion evaluation.
- Bumped the workspace version and visible install/status docs to 1.3.2.
- Split oversized JEPA/world-model source files below the repository file-size
  guard while preserving the existing module boundaries.
