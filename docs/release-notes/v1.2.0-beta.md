# v1.2.0-beta - Local World Model

v1.2.0-beta introduces the local world-model implementation branch on top of
the v1.1.0-beta.3 provider runtime and governed agent-evolution baseline.

## Added

- New `archon-world-model` crate for trace rows, labels, storage, advisor, counterfactual scoring, trainer gates, checkpoint metadata, and agent-evolution signals.
- `archon world` command family:
  - `status`
  - `ingest <session-id>`
  - `ingest --backfill`
  - `predict-next`
  - `score-actions`
  - `explain`
  - `record-outcome`
  - `train`
  - `trainer-tick`
  - `eval`
  - `promote`
  - `rollback`
- Cozo + JSONL corpus store under `~/.archon/world-model/`.
- Fail-open advisory contract for pipeline, provider runtime, memory surfacing, and agent evolution consumers.
- Runtime advisory ledger records for shell/TUI pipeline starts, memory reindex, observed provider-runtime starts, and governed agent-evolution commands.
- Runtime pipeline completions can attach actual outcomes, compute latent surprise, and link audited bundle metadata back to persisted predictions.
- CPU candidate training path with persisted candidate manifests, eval reports, and promotion blocked unless mandatory gates pass.
- CPU candidates now write Candle safetensors checkpoint files alongside JSON manifests.
- Model activation history for promote and rollback operations.
- Compact CPU latent transition model with auxiliary heads and deterministic test embedding adapter.
- Graph-aware transition features for session, agent, provider, plan, and memory-surfacing context.
- Redaction-before-embedding plus persistent cache rows for local and third-party world-model embeddings.
- External embedding policy ledger events for approved third-party world-model embedding calls.
- Provider-neutral heuristic/LLM/hybrid row labeler that works with any configured `LlmProvider`.
- Prediction records can now be updated with actual outcomes so Archon can compute latent surprise.
- k-NN counterfactual scoring with explicit non-causal framing and separate calibration reporting.
- Persisted counterfactual score records with neighbor evidence references.
- Idle-aware trainer gates and backend metadata for CPU, CUDA, and MLX Metal.
- Accelerator availability probes that run a tiny synchronized tensor operation before selecting CUDA or Metal.
- One-shot dynamic trainer tick that consumes stored rows and writes a candidate when row/surprise/correction/elapsed triggers fire.
- Background trainer ticks are scheduled after shell/TUI coding and research pipeline completions when idle and battery gates allow them.
- Governed agent evolution can use repeated world-model risk/surprise evidence during proposal generation, shadow evaluation, approval display, and reports.

## Dependency Baseline

- Includes the main-branch Dependabot remediation commit `406b31c`, which closes wasmtime, jsonwebtoken, serde-yaml, openssl, and git2 alerts.
- Workspace Rust version is inherited from main at `1.86`.

## Safety

- Advisor returns `None` on cold start, unavailable store, candidate-only state, training-in-progress, missing embeddings, or backend failure.
- Behavior-changing agent evolution remains shadow-evaluation and approval gated.
- Native CUDA and MLX Metal training/inference code paths remain feature/hardware-gated; CPU is the supported default. CUDA-feature validation passes locally on WSL, while MLX Metal still needs Apple Silicon validation.
- Raw JSONL ledgers rotate at 500 MB and retain 90 days by default; Cozo summaries remain.

## Validation

- Focused world-model unit suite covers ingestion, storage, labels, advisor fail-open behavior, counterfactual scoring, trainer gates, backend selection, and agent-evolution signals.
- CUDA-feature world-model library tests pass locally on WSL with `/usr/local/cuda-13.2` after driver/toolkit compatibility is corrected.
