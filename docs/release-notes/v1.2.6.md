# v1.2.6 - Symmetric Tool-Pair Sanitizers

v1.2.6 fixes the inverse of the v1.2.5 orphan `tool_result` repair. A
live resumed session exposed an assistant message that still contained
`tool_use` blocks whose ids were not covered by the immediately
following user message's `tool_result` blocks. Anthropic rejects that
shape before generation starts, so the patch repairs it at both the
source and provider boundary. The same release also repairs Codex
Responses replay shape so `function_call` and `function_call_output`
items stay coherent before the request leaves Archon.

## Fixed

- **Orphan `tool_use` sanitizer.** Anthropic-shape requests now demote
  uncovered assistant `tool_use` blocks to text, symmetric with
  v1.2.5's orphan `tool_result` demotion. Valid pairs are preserved,
  and partially covered assistant messages only demote the missing
  tool ids.
- **Shared message-invariants module.**
  `crates/archon-llm/src/message_invariants.rs` now owns the
  Anthropic-shape sanitizer used by Anthropic, Bedrock, and Vertex.
  Provider-local copies were removed so future pairing fixes land in
  one place.
- **Codex Responses pair repair.** The Codex translator now repairs
  OpenAI Responses input after translating Archon messages. Missing
  `function_call_output` items are synthesized with the upstream Codex
  `"aborted"` output text, orphan outputs are demoted to user text, and
  late matching outputs are moved back beside their `function_call`
  before the request is sent.
- **Atomic tool-result safety net.** Tool postprocessing now fills
  synthetic error `tool_result` blocks for any allowed tool id whose
  dispatch path did not record a result. This covers mid-turn
  cancellation, dispatch panics, and length mismatches that previously
  could persist an assistant `tool_use` without a matching user result.
- **Compaction boundary test alignment.** The context boundary test now
  expects `role: "user"`, matching the v1.2.5 boundary-role change and
  unblocking the Linux and macOS CI jobs.

## Notes
