# Archon v1.3.5 — Video Evidence Ingest Hardening

## Video Evidence

- Fixed TUI `/video ingest` CLI-mirror argument handling so flags such as
  `--frames`, `--asr`, `--metadata-only`, `--vlm`, and `--yes` reach the video
  ingest command unchanged.
- Fixed YouTube acquisition so governed `yt-dlp` downloads are used as the
  local media source for ASR and frame processing.
- `yt-dlp` output parsing now records the actual downloaded file path instead
  of the output template, and passes the Homebrew `ffmpeg` directory to
  `yt-dlp` when available.
- `whisper-cpp` ASR now reads `whisper-cli` JSON output and writes timecoded
  transcript chunks.
- Audio extraction now runs `ffmpeg` with `-nostdin -y`, preventing hidden TUI
  child processes from waiting on temporary WAV overwrite prompts.
- Frame extraction now writes PNG frames instead of MJPEG/JPEG, avoiding
  AV1/non-full-range YUV encoder failures on YouTube WebM downloads.
- `archon video reprocess <video-id> --frames` can retry frame extraction for a
  successful transcript/video ingest without duplicating transcript chunks.

## Setup And Documentation

- `scripts/install-system-deps.sh` now installs/checks video helper packages
  across supported OS families: `ffmpeg`, `ffprobe`, `yt-dlp`, and
  `whisper-cli` where packaged.
- `scripts/archon-init.sh` now creates `.archon/video-artifacts/downloads/` and
  seeds video policy defaults for new projects.
- Added a real-world cookbook for governed YouTube ingest with local
  `whisper-cpp`, including policy, TUI commands, monitoring, and failure
  recovery.
